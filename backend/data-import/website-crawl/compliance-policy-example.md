# 金融机构网络安全管理办法（征求意见稿）

## 政策背景
为加强金融机构网络安全管理，防范网络安全风险，维护金融稳定，根据《中华人民共和国网络安全法》、《中华人民共和国数据安全法》等法律法规，制定本办法。

## 征求意见截止日期
**2026年3月31日**

## 主要要求

### 1. 网络安全管理体系（第5-12条）
金融机构应当建立健全网络安全管理体系，包括：
- 设立网络安全委员会，由行长担任组长
- 制定网络安全管理制度和操作规程
- 建立网络安全责任制，明确各部门职责
- 定期开展网络安全风险评估和应急演练

### 2. 网络分区管理（第13-18条）
- **网络分区要求**: 金融机构应当实施网络分区管理，将内部网络划分为管理网域、业务网域、互联网网域
- **隔离要求**: 不同网域之间应当实施逻辑隔离，关键系统应当实施物理隔离
- **访问控制**: 建立跨网域访问审批机制，严格控制数据流向

### 3. 安全监测预警（第19-25条）
- **监测系统**: 部署网络安全监测系统，实现7×24小时实时监控
- **威胁情报**: 建立威胁情报收集和分析机制
- **预警响应**: 建立分级预警机制，明确响应流程和时限

### 4. 应急响应机制（第26-32条）
- **应急预案**: 制定网络安全事件应急预案，定期演练
- **应急处置**: 建立7×24小时应急响应团队
- **事件报告**: 重大网络安全事件应当在2小时内报告人民银行

### 5. 供应链安全管理（第33-38条）
- **供应商准入**: 建立信息技术供应商安全准入制度
- **合同约束**: 在合同中明确安全责任和义务
- **持续监督**: 定期评估供应商安全状况

## 实施要求
本办法自发布之日起**6个月后实施**（预计2026年7月），给予金融机构充足的准备时间。

## 适用范围
本办法适用于在中华人民共和国境内设立的银行业金融机构、证券公司、保险公司等金融机构。

## 意见反馈方式
各金融机构应当于2026年3月31日前将书面意见反馈至人民银行金融科技司。
