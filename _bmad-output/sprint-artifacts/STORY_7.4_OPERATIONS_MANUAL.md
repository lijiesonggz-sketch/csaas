## 功能概述

### 系统目标

AI成本优化工具旨在帮助管理员：
- **监控** AI使用成本和趋势
- **识别** 高成本客户和低效使用模式
- **优化** AI资源配置以降低成本
- **预警** 成本超标和异常情况
- **报告** 成本数据用于决策分析

### 核心功能模块

#### 1. 成本监控仪表板
- 实时显示总成本、平均成本、高成本客户
- 可视化成本趋势图表
- 按任务类型分类的成本分析

#### 2. 优化建议引擎
- 自动识别成本优化机会
- 提供具体的优化建议和预期节省
- 支持批量应用优化措施

#### 3. 告警系统
- 成本超标自动告警
- 异常成本波动检测
- 多级告警通知机制

#### 4. 报告导出
- CSV/Excel格式导出
- 自定义时间范围
- 按组织筛选数据

## 使用指南

### 访问成本优化页面

1. **登录系统**
   - 使用管理员账号登录系统
   - 只有管理员角色可以访问成本优化功能

2. **导航到成本优化页面**
   - 点击左侧菜单 "管理中心" → "AI成本优化"
   - 或直接访问 `/admin/cost-optimization`

### 查看成本指标

#### 总览卡片

页面顶部显示三个关键指标卡片：

1. **总成本 (Total Cost)**
   - 显示当前月份的AI使用总成本（人民币）
   - 包含所有组织的累计成本
   - 实时更新

2. **平均成本 (Average Cost)**
   - 显示每个组织的平均成本
   - 计算公式：总成本 ÷ 组织数量
   - 用于评估整体成本水平

3. **高成本客户数 (High Cost Clients)**
   - 显示超过阈值（500元/月）的组织数量
   - 点击可查看详细列表
   - 红色标识表示需要关注

#### 成本趋势图

**功能说明：**
- 显示最近30天的每日成本趋势
- 折线图展示成本变化趋势
- 鼠标悬停查看具体日期和金额

**如何使用：**
1. 观察趋势线的走向（上升/下降/平稳）
2. 识别异常峰值或突然下降
3. 点击图表可查看详细数据点

**时间范围调整：**
- 默认显示30天
- 可通过下拉菜单选择7天、30天、90天
- 自定义日期范围（点击"自定义"按钮）

### 管理高成本客户

#### 查看高成本客户列表

**列表信息包括：**
- 组织名称
- 当月总成本
- AI调用次数
- 主要使用的任务类型
- 成本趋势（上升/下降）

**排序和筛选：**
- 默认按成本从高到低排序
- 点击列标题可切换排序方式
- 使用搜索框快速查找特定组织

#### 查看组织详情

1. **点击组织卡片**
   - 打开详情对话框
   - 显示完整的成本分析

2. **详情内容：**
   - **成本分解：** 按任务类型分类的成本占比
   - **使用模型：** 各AI模型的使用频率和成本
   - **时间分布：** 按小时/天的使用模式
   - **历史趋势：** 最近3个月的成本变化

3. **可执行操作：**
   - 查看详细使用日志
   - 应用优化建议
   - 设置自定义阈值
   - 导出组织报告

### 使用优化建议

#### 查看建议列表

**建议类型：**

1. **切换模型 (Switch Model)**
   - **触发条件：** 使用昂贵模型处理简单任务
   - **建议内容：** 将GPT-4切换为Qwen-Plus
   - **预期节省：** 约40-60%成本
   - **影响评估：** 轻微质量下降，但仍满足需求

2. **降低频率 (Reduce Frequency)**
   - **触发条件：** 每日调用次数>100次
   - **建议内容：** 将每日推送改为每周推送
   - **预期节省：** 约70%成本
   - **影响评估：** 信息时效性降低

3. **优化提示词 (Optimize Prompts)**
   - **触发条件：** 平均token数过高
   - **建议内容：** 精简提示词，减少不必要的上下文
   - **预期节省：** 约20-30%成本
   - **影响评估：** 需要测试以确保质量

4. **结果缓存 (Cache Results)**
   - **触发条件：** 重复查询相同内容
   - **建议内容：** 实施结果缓存机制
   - **预期节省：** 约30-50%成本
   - **影响评估：** 需要开发工作

#### 应用单个建议

1. **点击建议卡片** 查看详细信息
2. **评估影响** 查看受影响的组织列表
3. **点击"应用建议"按钮**
4. **确认操作** 在弹出对话框中确认
5. **等待执行** 系统自动应用优化措施
6. **查看结果** 显示执行成功/失败状态

#### 批量优化操作

**使用场景：**
- 需要对多个组织应用相同的优化措施
- 统一调整成本控制策略
- 快速响应成本超标情况

**操作步骤：**

1. **选择组织**
   - 在高成本客户列表中勾选目标组织
   - 可使用"全选"快速选择所有组织
   - 已选数量显示在页面顶部

2. **点击"批量优化"按钮**
   - 打开批量优化对话框
   - 显示已选组织列表

3. **选择优化操作**
   - 从下拉菜单选择操作类型：
     - 切换到经济模型
     - 降低分析频率
     - 暂停非关键功能
   - 填写操作说明（可选）

4. **预览影响**
   - 查看预期节省金额
   - 查看可能的服务影响
   - 确认受影响的功能

5. **执行批量操作**
   - 点击"确认执行"按钮
   - 系统逐个处理选中的组织
   - 显示实时进度条

6. **查看执行结果**
   - 成功数量
   - 失败数量及原因
   - 详细的执行日志

**注意事项：**
- 批量操作不可撤销，请谨慎确认
- 建议先在单个组织测试效果
- 操作会记录在审计日志中
- 失败的操作不会影响其他组织

### 导出成本报告

#### 快速导出

1. **点击"导出报告"按钮**
   - 位于页面右上角
   - 默认导出当月数据为CSV格式

2. **等待下载**
   - 浏览器自动下载文件
   - 文件名格式：`cost-report-YYYY-MM-DD.csv`

#### 自定义导出

1. **点击"导出选项"按钮**
   - 打开导出配置对话框

2. **选择导出格式**
   - **CSV：** 简单表格，适合数据分析
   - **Excel：** 多工作表，包含图表

3. **设置时间范围**
   - 选择开始日期和结束日期
   - 或使用快捷选项（本月、上月、本季度）

4. **筛选组织（可选）**
   - 留空导出所有组织
   - 或选择特定组织

5. **点击"导出"按钮**
   - 系统生成报告文件
   - 自动下载到本地

#### 报告内容说明

**CSV报告包含：**
- 日期、组织ID、组织名称
- 任务类型、AI模型
- 输入token数、输出token数
- 成本金额、调用时长

**Excel报告包含：**
- **概览表：** 总成本、平均成本、趋势图
- **组织明细表：** 每个组织的详细数据
- **任务类型表：** 按任务类型分类统计
- **模型使用表：** 各AI模型的使用情况
- **趋势图表：** 可视化图表

## 告警处理流程

### 告警类型

#### 1. 成本超标告警 (Cost Exceeded)

**触发条件：**
- 组织当月成本超过500元阈值

**告警级别：** 高 (High)

**处理流程：**

1. **接收告警**
   - 系统自动发送站内通知
   - 邮件通知管理员（如已配置）
   - 仪表板显示红色告警标识

2. **查看详情**
   - 点击告警消息查看详细信息
   - 确认超标金额和超标比例
   - 查看成本构成分析

3. **分析原因**
   - 检查是否有异常使用模式
   - 确认是否为正常业务增长
   - 查看最近的配置变更

4. **采取措施**
   - **立即措施：**
     - 切换到经济模型（Qwen-Plus）
     - 降低非关键功能的分析频率
     - 暂停低ROI的推送任务

   - **中期措施：**
     - 与客户沟通，调整服务级别
     - 优化提示词以减少token消耗
     - 实施结果缓存机制

   - **长期措施：**
     - 调整定价策略
     - 升级客户套餐
     - 优化AI使用架构

5. **确认告警**
   - 点击"确认"按钮标记告警已处理
   - 填写处理说明
   - 告警状态变更为"已确认"

6. **持续监控**
   - 观察优化措施的效果
   - 跟踪后续成本变化
   - 必要时进一步调整

#### 2. 成本激增告警 (Cost Spike)

**触发条件：**
- 单日成本较前一日增长>50%

**告警级别：** 中 (Medium)

**处理流程：**

1. **快速响应**
   - 立即查看告警详情
   - 确认是否为系统异常

2. **排查原因**
   - 检查是否有批量任务执行
   - 确认是否有新客户上线
   - 查看错误日志是否有重试风暴

3. **紧急处理**
   - 如果是异常：立即停止相关任务
   - 如果是正常：标记为已知并监控
   - 通知相关技术团队

4. **预防措施**
   - 设置单日成本上限
   - 实施熔断机制
   - 优化重试策略

#### 3. 模型降级告警 (Model Degradation)

**触发条件：**
- 频繁降级到备用模型（>30%请求）

**告警级别：** 低 (Low)

**处理流程：**

1. **评估影响**
   - 检查服务质量指标
   - 查看用户反馈

2. **分析原因**
   - 主模型API是否稳定
   - 是否达到速率限制
   - 网络连接是否正常

3. **解决方案**
   - 联系AI服务提供商
   - 调整速率限制配置
   - 考虑增加API配额

### 告警管理最佳实践

**1. 定期检查告警**
- 每日至少检查一次告警列表
- 优先处理高级别告警
- 及时确认已处理的告警

**2. 建立响应SLA**
- 高级别告警：2小时内响应
- 中级别告警：8小时内响应
- 低级别告警：24小时内响应

**3. 记录处理过程**
- 详细记录问题原因
- 记录采取的措施
- 记录处理结果和效果

**4. 定期回顾**
- 每周回顾告警趋势
- 识别重复出现的问题
- 优化告警阈值设置

## 常见问题解答

### Q1: 为什么我的成本突然增加？

**可能原因：**
1. 新客户上线或业务增长
2. 分析频率增加（如从周报改为日报）
3. 使用了更昂贵的AI模型
4. 系统错误导致重复调用

**排查步骤：**
1. 查看成本趋势图，确定增长时间点
2. 检查该时间段的配置变更
3. 查看使用日志，识别异常模式
4. 对比不同组织的成本变化

**解决方案：**
- 如果是正常增长：调整预算或优化配置
- 如果是异常：回滚配置或修复错误

### Q2: 如何降低AI使用成本？

**立即见效的方法：**
1. **切换到经济模型**
   - GPT-4 → Qwen-Plus（节省约50%）
   - Claude-3 → Qwen-Turbo（节省约60%）

2. **降低分析频率**
   - 技术雷达：每日 → 每周（节省约70%）
   - 行业雷达：实时 → 每日（节省约50%）

3. **优化提示词**
   - 移除不必要的示例
   - 精简系统提示
   - 使用更简洁的输出格式

**中长期优化：**
1. 实施智能缓存机制
2. 批量处理相似请求
3. 使用更精准的触发条件
4. 定期清理低价值推送

### Q3: 批量优化会影响服务质量吗？

**影响评估：**

**切换模型：**
- 轻微影响：分析准确度可能下降5-10%
- 建议：先在少数组织测试，观察反馈

**降低频率：**
- 中等影响：信息时效性降低
- 建议：保持合规雷达的高频率，降低其他类型

**暂停功能：**
- 显著影响：部分功能不可用
- 建议：仅在紧急情况使用，并提前通知客户

**最佳实践：**
1. 始终先在测试组织验证
2. 逐步推广，观察效果
3. 保持与客户的沟通
4. 准备回滚方案

### Q4: 导出的报告数据不准确怎么办？

**检查清单：**
1. 确认选择的时间范围是否正确
2. 检查是否有筛选条件限制
3. 验证数据库中的原始数据
4. 确认是否有数据同步延迟

**解决步骤：**
1. 重新导出报告，对比结果
2. 查看系统日志，检查错误信息
3. 联系技术支持，提供详细信息
4. 如果是系统问题，等待修复后重新导出

### Q5: 如何设置自定义成本阈值？

**当前版本：**
- 默认阈值：500元/月/组织
- 暂不支持自定义阈值（计划在未来版本实现）

**临时方案：**
1. 使用导出功能定期检查成本
2. 手动标记需要关注的组织
3. 设置日历提醒定期检查

**未来功能：**
- 组织级别自定义阈值
- 多级阈值告警
- 预算管理功能

### Q6: 优化建议是如何生成的？

**分析算法：**

1. **高成本识别**
   - 当月成本 > 500元
   - 成本增长率 > 50%

2. **低效模式检测**
   - 使用GPT-4处理简单任务
   - 单日调用次数 > 100次
   - 平均token数 > 阈值

3. **ROI分析**
   - 推送内容的用户反馈评分
   - 内容的实际查看率
   - 成本与价值的比率

4. **节省估算**
   - 基于历史数据计算
   - 考虑模型价格差异
   - 预估实施影响

**更新频率：**
- 每日凌晨自动分析
- 手动刷新可立即重新分析

## 故障排查

### 问题1: 页面加载缓慢或超时

**症状：**
- 成本优化页面打开超过10秒
- 图表加载失败
- 请求超时错误

**可能原因：**
1. 数据量过大（历史数据过多）
2. 数据库查询性能问题
3. 网络连接不稳定
4. 服务器资源不足

**排查步骤：**

1. **检查网络连接**
   ```bash
   # 测试API连接
   curl -I https://your-domain.com/api/v1/admin/cost-optimization/metrics
   ```

2. **查看浏览器控制台**
   - 打开开发者工具（F12）
   - 查看Network标签，识别慢请求
   - 查看Console标签，检查JavaScript错误

3. **检查服务器日志**
   ```bash
   # 查看后端日志
   tail -f backend/logs/application.log | grep "cost-optimization"
   ```

4. **检查数据库性能**
   ```sql
   -- 查看慢查询
   SELECT * FROM pg_stat_statements
   WHERE query LIKE '%ai_usage_logs%'
   ORDER BY mean_exec_time DESC
   LIMIT 10;
   ```

**解决方案：**

1. **优化数据查询**
   - 添加数据库索引
   - 限制查询时间范围
   - 使用分页加载

2. **清理历史数据**
   ```bash
   # 归档90天前的数据
   npm run archive-old-logs
   ```

3. **增加服务器资源**
   - 扩展数据库连接池
   - 增加Redis缓存
   - 升级服务器配置

4. **启用缓存**
   - 确认Redis服务正常运行
   - 检查缓存配置
   - 清除过期缓存

### 问题2: 批量优化操作失败

**症状：**
- 批量操作提交后无响应
- 部分组织优化失败
- 错误消息："Operation failed"

**可能原因：**
1. 数据库事务冲突
2. 组织配置不存在
3. 权限验证失败
4. 并发操作限制

**排查步骤：**

1. **查看详细错误信息**
   - 点击失败的组织查看错误详情
   - 记录错误代码和消息

2. **检查组织状态**
   ```bash
   # 查询组织配置
   npm run check-org-config <organizationId>
   ```

3. **查看操作日志**
   ```bash
   # 查看最近的优化操作
   tail -f backend/logs/optimization.log
   ```

4. **验证权限**
   - 确认当前用户是管理员角色
   - 检查JWT token是否有效

**解决方案：**

1. **重试失败的操作**
   - 单独处理失败的组织
   - 检查并修复数据问题后重试

2. **分批处理**
   - 减少单次操作的组织数量
   - 每批不超过10个组织

3. **修复数据问题**
   ```bash
   # 修复缺失的配置
   npm run fix-org-configs
   ```

4. **联系技术支持**
   - 提供错误日志
   - 说明操作步骤
   - 提供受影响的组织ID

### 问题3: 导出报告失败

**症状：**
- 点击导出按钮无反应
