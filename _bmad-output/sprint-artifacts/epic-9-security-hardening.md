# Epic 9: 安全加固与认证增强

## 背景

基于 PRD 与代码实现差距分析，发现系统存在以下安全隐患需要修复：

1. **JWT Secret 使用默认值** - 生产环境可能使用不安全的默认密钥
2. **JWT 会话超时过长** - 当前 7 天超时时间过长，不符合安全最佳实践
3. **缺少登录失败锁定** - 暴力破解攻击风险

## 目标

修复上述安全隐患，确保系统认证机制符合安全最佳实践。

## 包含 Stories

- 9-1-jwt-security-hardening: JWT Secret 和会话超时加固
- 9-2-login-failure-lockout: 登录失败锁定机制

## 成功标准

- JWT Secret 不再使用默认值，启动时强制校验
- JWT 会话超时缩短至 2 小时
- 连续 5 次登录失败后账户锁定 30 分钟
- 所有现有测试通过
- 新增安全相关单元测试
