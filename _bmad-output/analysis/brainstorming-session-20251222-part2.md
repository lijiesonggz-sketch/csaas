## 阶段2：SCAMPER Method - Substitute（替代）视角执行结果

**技术焦点：** 系统性探索更好的替代方案优化设计

### 核心突破：多模型协同架构

**传统方案的问题：**
- 单一AI模型存在幻觉、不一致、质量难验证
- 纯人工审核效率低，难以规模化
- 置信度标注缺乏客观依据

**创新替代方案：三模型众包+互审机制**

#### 架构1：生成环节（综述、聚类、矩阵、问卷、落地措施）

**设计：3模型并行独立生成+交叉验证**
```
模型1（如GPT-4）、模型2（如Claude）、模型3（如国产模型）
  ↓ 并行独立生成
结果1、结果2、结果3
  ↓ 交叉对比
计算两两相似度（条目级对比/语义相似度/结构化评分）
  ↓ 质量判定
平均相似度 ≥ 80% → 采纳（投票选最佳）
平均相似度 < 80% → 诊断性重试
```

**相似度计算方法：**
- 聚类环节：条目级对比（重合度）
- 综述生成：语义embedding相似度
- 矩阵生成：结构化单元格对比

**失败处理机制（混合模式）：**
1. 第一次<80% → 诊断差异原因（信息不足/逻辑矛盾/理解偏差）
2. 诊断性重试1次（补充材料/调整提示/换模型）
3. 仍不通过 → 降级为"初稿"状态，高亮提示主咨询师"需要额外关注"
4. 主咨询师选择：接受并手工修改/要求重新生成/完全手工处理

#### 架构2：评分环节（差距分析打分）

**设计：3模型独立评分+差异检测**
```
模型1、模型2、模型3 独立对每个条目打分
  ↓
每条目平均分 = (分1+分2+分3)/3
  ↓
相对差异 = max|分x - 平均分| / 平均分
  ↓
差异 > 40% → 高亮标记"三模型评分存在显著分歧"
差异 ≤ 40% → 正常显示平均分
```

**差异阈值设计：**
- 采用相对差异（40%）而非绝对差异
- 示例（5分制）：
  - 平均3分，差异>1.2分触发高亮
  - 平均4.5分，差异>1.8分触发高亮
- 可根据实际运行调整或采用分段阈值

**高亮后处理：**
- 显示三个模型的具体分数
- AI分析可能的分歧原因
- 主咨询师可选择：采用平均分/采用某个模型分数/手工调整

### 核心优势

1. **质量保障** - 三个模型达成共识（80%相似）才采纳，大幅降低单点偏差
2. **客观置信度** - 基于模型一致性而非自评，更可靠
3. **智能容错** - 诊断性重试针对性解决问题，降级机制保证流程不卡死
4. **差异透明** - 评分分歧高亮显示，帮助主咨询师识别争议点
5. **民主决策** - 众包模式发挥各模型长处

### 成本分析

**单项目API调用：**
- 生成环节：约19次（3模型×5环节 + 20%重试率）
- 评分环节：3次
- 总计：约22次调用

**成本估算：**
- 约$1/项目（≈7元人民币）
- 占咨询项目收费比例：<0.01%
- **结论：质量第一，成本可忽略不计**

### Demo阶段实施要点

**必须实现：**
- 三模型并行调用接口
- 相似度计算算法（可先用简化版语义相似度）
- 投票选最佳结果逻辑（建议简单多数）
- 差异检测和高亮UI
- 降级到"初稿"状态的界面设计

**可简化或后期优化：**
- 诊断性重试（Demo可简化为直接降级）
- 分段差异阈值（先用统一40%）
- 融合生成（先用简单投票）

## 阶段3：Assumption Reversal（假设反转）执行结果

**技术焦点：** 挑战核心假设，发现隐藏风险和盲点

### 识别的核心假设

**关于AI能力：**
1. 大模型能够理解IT标准文档并准确提炼要点
2. 三个模型达成共识（80%相似）= 结果可靠
3. AI生成的问卷质量足够专业，咨询师只需审核不需大改

**关于人机协作：**
4. 主咨询师愿意花半天时间审核每个环节
5. 主咨询师能够发现AI的错误（而不是被AI误导）✅ 最担心
6. 咨询师会信任AI的分析，而不是觉得"AI抢了我的饭碗"

**关于商业模式：**
7. 降低成本50%就能吸引企业客户
8. 咨询公司愿意用系统（而不是担心被平台化）
9. 数据积累到一定量就能形成壁垒

**关于技术架构：**
10. 多模型调用的成本和延迟是可接受的

### 重点反转1：三个模型达成共识 ≠ 结果一定正确

**极端场景：**
- 标准文档本身存在模糊或歧义表述
- 三个模型都基于同样的误解，做出一致但错误的判断
- 系统显示"高置信度"，主咨询师因此降低警惕
- 错误结果被采纳并固化

**应对策略（人工抽查机制）：**

**智能抽查优先级：**
```
高优先级抽查（必查）：
- 新的咨询领域第一次使用
- AI置信度<75%的结果
- 三模型分歧较大的条目
- 涉及合规或法律的条款

低优先级抽查（随机10%）：
- 高置信度(>85%)的常规条目
```

**抽查执行：**
- Demo阶段：创始人100%审核前10个项目
- 成长期：随机抽查30%（10-50个项目）
- 成熟期：抽查10% + 自动化质量监控

**外部验证：**
- 定期请行业专家审核系统生成结果
- 建立"上帝视角"质量基准

### 重点反转2：主咨询师可能被AI误导（最担心）✅

**极端场景：**
- AI生成的结果"看起来很专业"，用词准确，逻辑自洽
- 主咨询师因时间有限、对标准不够熟悉、过度信任"高置信度"标签
- 审核流于形式，AI错误被固化
- 认知偏差：自动化偏见、锚定效应、确认偏差

**应对策略（渐进式披露）：**

#### 三阶段审核流程设计

**阶段1：独立思考阶段**
- 主咨询师必须先查看原始标准文档
- 可选：记录自己的初步判断（不强制）
- 目标：激活咨询师的独立思考，避免被动接受

**阶段2：对比发现阶段**
- 展示AI三模型分析结果 + 一致度
- 如果咨询师填写了初步判断：自动对比差异并高亮
- 明确标注高风险项（置信度低、三模型分歧、涉及合规）
- 提供选项：接受全部/重点审核/完全重做

**阶段3：深度验证阶段（针对高风险项）**
- 展示AI的聚类逻辑说明
- 原文对照显示
- 提供挑战性问题引导思考
- 让咨询师主动决策而非被动接受

#### 四个防止流于形式的机制

**机制1：审核质量追踪**
```
追踪指标：
- 平均审核时间（<10分钟/环节 → 预警）
- 修改率（0% → 可能过度依赖）
- 高风险项审核率

预警触发：
连续3次快速通过 + 零修改 + 高风险项全接受
→ 提示重新审核或说明理由
```

**机制2：定期能力校准测试**
- 每月混入1-2个AI故意生成的错误结果
- 咨询师能发现 → 能力合格 ✅
- 咨询师未发现 → 触发培训提醒 ⚠️

**机制3：同行互审（可选）**
- 重要项目由两个主咨询师交叉审核
- 意见一致 → 高置信度
- 意见分歧 → 需要仲裁或深度讨论

**机制4：创始人终极审核**
- Demo和早期：创始人作为"上帝视角"
- 渐进式降低审核比例：100% → 30% → 10%
- 建立质量baseline和监控体系

### 重点反转3：咨询公司可能内部抵制

**极端场景：**
- 管理层决定采用系统（降本增效）
- 一线咨询师强烈抵制（担心被替代、失去控制权、为平台打工）
- 消极抵抗形式：不认真审核、技术性抵制、暗示客户质量下降、数据投毒

**应对策略（赋能定位）：**

**1. 营销话术调整**
- ❌ "AI替代人工提高效率"
- ✅ "AI让咨询师能力提升10倍，服务10倍客户"
- ✅ "培养超级个体咨询师，而非工具人"

**2. 产品中的赋能设计**

**个人品牌建设：**
- 每个咨询师有专家主页
- 显示：完成项目数、客户评分、擅长领域
- 平台认证专家标识

**知识产权保护：**
- 咨询师优化的内容可标记为"专家版本"
- 其他人使用需要署名或分成
- 让咨询师的经验在平台上有价值

**透明度控制权：**
咨询师决定客户看到什么：
- 完全黑盒："专家咨询报告"
- 半透明："AI辅助的专家分析"
- 完全透明："AI生成+专家审核"

**3. 渐进式导入策略**
- 第一阶段：只用AI做标准检索和综述（最无聊的活）
- 第二阶段：逐步扩展到聚类、问卷
- 让咨询师逐渐习惯和信任

### Assumption Reversal 核心洞察

**关键发现：**
1. **技术风险可控** - 三模型共识机制 + 创始人专业能力 = 双重保障
2. **人的因素最关键** - 防止咨询师被AI误导是最大挑战
3. **组织变革可管理** - 合作方是咨询老大 + 赋能定位 + 创始人能力 = 可推动

**最优先应对：**
1. ✅ 渐进式披露三阶段审核流程（防误导）
2. ✅ 智能抽查机制（质量兜底）
3. ✅ 赋能定位而非替代定位（减少抵抗）

**Demo阶段关键：**
- 创始人作为第一个主咨询师，亲自验证系统质量
- 用实际效果说服合作方，而非纸上谈兵
- 建立质量基准和最佳实践
