# 项目范围界定与分阶段发展路线

> **关联文档**: [产品需求文档 (PRD)](./prd.md)

## MVP 战略与哲学

**选定方向:体验型 MVP(Experience MVP)**

**核心理念:**
Csaas 的价值主张是"将2-3周的咨询项目缩短至1周",这不是单点工具优化,而是**完整流程体验的重构**。DEMO阶段必须让咨询师体验到端到端的自动化价值,才能验证核心假设:"AI辅助能否真正让咨询师从重复劳动中解放"。

**MVP 聚焦策略:**
- ✅ **做完整但精简的流程** - 覆盖项目启动→AI生成→人工审核→问卷收集→报告交付的完整闭环
- ✅ **做最痛的3个环节** - 聚类、问卷、落地措施是咨询师最耗时且最容易出错的部分
- ❌ **不做锦上添花的功能** - 对标数据、专家优化版等增值服务留到有真实数据后再做

**资源预算:**
- **团队规模**:4-6人(1前端+2后端+1AI工程师+1产品经理+可选QA)
- **开发周期**:4个月
- **目标里程碑**:完成前10个真实项目的端到端验证

## MVP 功能集(Phase 1:DEMO阶段,0-4个月)

### 核心用户旅程支持

**主咨询师李明的关键旅程:**
1. ✅ 上传IT标准文档(ISO 27001 PDF,200页)
2. ✅ 选择咨询领域配置(如"信息安全管理体系")
3. ✅ 触发AI三模型并行生成聚类
4. ✅ 审核聚类结果(渐进式披露界面:查看AI逻辑说明→对比三模型一致性→深度验证高风险项)
5. ✅ 触发AI生成问卷题目
6. ✅ 审核并定制问卷(调整题目措辞、删除/新增题目)
7. ✅ 发送问卷给企业方(邮件分发+提醒)
8. ✅ 触发AI生成落地措施(基于问卷结果的差距分析)
9. ✅ 审核落地措施并导出PDF报告
10. ✅ 交付给客户

**企业PM张经理的关键旅程:**
1. ✅ 收到主咨询师分享的问卷链接
2. ✅ 查看项目进度(多少人已填写、完成率)
3. ✅ 催办未填写成员
4. ✅ 下载差距分析报告

**被调研者王工的关键旅程:**
1. ✅ 收到邮件邀请填写问卷
2. ✅ 在线填写问卷(支持保存草稿、续填)
3. ✅ 提交后查看自己的答案

**平台管理员小刘的关键旅程(简化版):**
1. ✅ 监控AI任务执行状态(成功率、失败原因)
2. ⚠️ 手工上传/更新IT标准库(DEMO阶段无自动化)
3. ❌ 暂不支持:审核专家优化版、配置对标数据

### 必须实现的核心能力

**1. AI 生成引擎(核心3环节)**

**环节1:聚类分析**
- **输入**:IT标准文档(PDF/Word,200-300页)
- **输出**:聚类结果(如ISO 27001的114条要求聚类为12-15个类别)
- **三模型验证**:
  - 结构一致性 ≥ 90%(类别数量±1、条款归属重合度 ≥ 85%)
  - 语义等价性 ≥ 80%(类别命名的语义相似度)
  - 细节一致性 ≥ 60%(条款排序、描述措辞)

**环节2:问卷生成**
- **输入**:聚类结果 + 咨询领域配置
- **输出**:结构化问卷(50-100题,包含题目、选项、评分规则)
- **三模型验证**:
  - 覆盖率检查(每个聚类至少3-5题)
  - 题目逻辑一致性(无重复、无矛盾)

**环节3:落地措施生成**
- **输入**:问卷结果(企业自评分) + 标准要求
- **输出**:差距分析 + 优先级排序的改进建议(20-30条)
- **三模型验证**:
  - 差距识别准确性(低分项是否被识别)
  - 建议可行性(是否符合企业实际)

**2. 分层相似度验证算法**

- **结构层验证**(90%阈值):
  - 聚类环节:条目级集合对比(Jaccard相似度) + 匈牙利算法最优匹配
  - 问卷环节:题目数量对比 + 聚类覆盖度对比

- **语义层验证**(80%阈值):
  - 使用 OpenAI text-embedding-ada-002 计算语义向量
  - 余弦相似度 ≥ 0.80

- **细节层验证**(60%阈值):
  - 编辑距离 / 最长公共子序列(LCS)

**3. 人工审核界面(渐进式披露)**

**阶段1:独立思考(防止锚定偏差)**
- 展示原始标准文档片段
- 可选:咨询师记录初步判断

**阶段2:对比发现**
- 并排展示三模型结果 + 一致度评分
- 高亮显示:
  - 🔴 高风险项(置信度<75%、三模型分歧>40%、涉及合规条款)
  - 🟡 中等风险项(置信度75-85%)
  - 🟢 高置信度项(置信度≥85%)
- 提供选项:全部接受 / 重点审核 / 完全重做

**阶段3:深度验证(针对高风险项)**
- AI逻辑说明:"为什么将条款A和B聚在一起?共同点是..."
- 原文对照显示
- 挑战性问题引导:"这两条是否真的属于同一类别?"

**4. 问卷收集系统**

- 邮件分发(集成 SendGrid / 阿里云邮件推送)
- 在线问卷填写界面(支持草稿保存、断点续填)
- 进度追踪(已填写人数、完成率)
- 催办提醒(自动/手动)

**5. 报告生成与导出**

- 差距分析报告(PDF格式):
  - 聚类维度成熟度评分(雷达图)
  - 低分项详细说明
  - 优先级排序的落地措施
- Excel导出(问卷原始数据、评分明细)

**6. 基础权限管理(RBAC)**

- **主咨询师**:创建项目、上传标准、审核AI结果、发送问卷、导出报告
- **企业PM**:查看项目进度、催办问卷、下载报告(无法看到个人问卷答案)
- **被调研者**:填写问卷、查看自己的答案
- ⚠️ **DEMO阶段简化**:暂无平台管理员复杂权限(手工配置即可)

**7. 单租户部署**

- DEMO阶段无需多租户隔离(单一客户/测试环境)
- 数据库:PostgreSQL 单实例
- 无需租户ID字段

## 风险缓解策略

### 技术风险

**风险1:仅3环节是否足够验证三模型架构可行性?**
- **分析**:聚类是最复杂的环节(涉及条目级匹配、图同构算法),如果聚类验证成功,其他环节技术风险更低
- **缓解策略**:
  - 优先实现聚类环节的完整验证逻辑
  - 在前10个项目中密切跟踪AI准确率
  - 如聚类成功率 ≥ 80%,证明架构可行

**风险2:三模型API稳定性和成本**
- **分析**:依赖OpenAI、Anthropic、国产模型API,任何一家故障影响系统可用性
- **缓解策略**:
  - 实现多级降级(见 technical-requirements.md ADR #2)
  - 单模型超时60秒 → 继续等待其他2个
  - 2个模型成功 → 提高阈值到90%继续使用
  - 监控每月API成本,设置预算告警

**风险3:相似度算法误判**
- **分析**:分层相似度验证可能将"实质不同但表述相似"的结果判为一致
- **缓解策略**:
  - 建立"AI错误案例库",记录所有被人工纠正的错误
  - 每完成10个项目,回顾错误模式并调整阈值
  - 初期采用保守阈值(结构95%、语义85%),逐步优化

### 市场风险

**风险4:缺少综述和矩阵,咨询师觉得功能不完整?**
- **分析**:综述和矩阵相对简单(综述是摘要,矩阵是表格填写),咨询师可能认为"不值得为3个环节付费"
- **缓解策略**:
  - 定价策略:DEMO阶段按项目收费(¥2,000-3,000/项目),而非按功能计费
  - 营销话术:强调"最痛的3个环节已自动化,综述和矩阵您10分钟就能手工完成"
  - 承诺路线图:明确告知"综述和矩阵将在3个月后上线"

**风险5:无对标数据,企业客户觉得报告不够专业?**
- **分析**:企业希望看到"您的成熟度在行业中处于XX分位",纯自评分说服力不足
- **缓解策略**:
  - 提供静态参考数据:"根据公开行业报告,同类企业平均分为3.2/5"
  - 承诺数据积累:"预计6个月后提供真实对标数据"
  - 聚焦差距分析而非排名:"重点是找到改进方向,而非攀比"

### 资源风险

**风险6:4个月能否完成开发?**
- **分析**:3个AI环节 + 相似度验证 + 审核界面 + 问卷系统,工作量不小
- **缓解策略**:
  - 采用成熟技术栈(React/Vue + NestJS/Fastify + Prisma + PostgreSQL)
  - 使用现成UI组件库(Ant Design / Element Plus)
  - 问卷系统可集成第三方(如金数据API),无需从零开发
  - 设定最小可用标准:前10个项目由创始人手工辅助完成,积累真实反馈

**风险7:团队规模不足(少于6人)怎么办?**
- **分析**:如果只有3-4人团队,4个月可能做不完
- **缓解策略**:
  - **最小化方案**:先实现1个环节(聚类) + 人工审核界面,验证核心技术可行性(2个月)
  - 其他环节用"半自动化"替代:AI生成但不做三模型验证,直接交给咨询师审核
  - 延长DEMO周期至5-6个月

## 分阶段发展路线图

### Phase 1:DEMO阶段(0-4个月) - MVP验证

**里程碑:**
- **M1(1个月)**:基础架构 + 聚类环节(单模型)
- **M2(2个月)**:三模型并行 + 相似度验证
- **M3(3个月)**:问卷生成 + 落地措施生成
- **M4(4个月)**:人工审核界面 + 问卷收集系统

**验证指标:**
- ✅ 完成前10个真实项目
- ✅ AI聚类准确率 ≥ 80%(咨询师修改<20%内容)
- ✅ 审核总时间 ≤ 3小时/项目
- ✅ 咨询师满意度 ≥ 4/5
- ✅ 系统可用性(非降级模式)≥ 85%

**成功标准:**
- 至少5位咨询师愿意推荐给同行
- 至少3位咨询师表示"愿意付费订阅"

---

### Phase 2:Growth阶段(5-12个月) - 规模化

**新增功能:**
- 综述生成环节
- 矩阵生成环节
- 对标数据库V1(基础百分位排名)
- 多租户架构(支持5-10家咨询公司)
- 订阅计费系统

**验证指标:**
- ✅ 完成100个项目(10家咨询公司 × 平均10个项目)
- ✅ 月活咨询师 ≥ 30人
- ✅ 订阅续约率 ≥ 70%
- ✅ 对标数据覆盖3个行业(金融、制造、医疗)

**成功标准:**
- 实现单月收入10万元(订阅 + 项目增值服务)
- 数据飞轮开始显现价值(用户反馈"对标数据很有用")

---

### Phase 3:Expansion阶段(13-24个月) - 壁垒构建

**新增功能:**
- 专家优化版系统
- 企业SSO集成
- 高级数据分析(AI质量趋势、成本优化)
- API开放平台
- 移动端优化

**验证指标:**
- ✅ 完成1000个项目
- ✅ 月活咨询师 ≥ 150人
- ✅ 认证专家 ≥ 50人
- ✅ 对标数据覆盖10个行业
- ✅ 订阅续约率 ≥ 80%

**成功标准:**
- 实现月收入50万元
- 竞争对手追赶需要2-3年数据积累
- 成为IT咨询行业事实标准工具
