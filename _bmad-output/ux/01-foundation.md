## Step 2: Project Understanding

### 项目本质

Csaas是一个**AI驱动的IT咨询成熟度评估SaaS平台**，通过三模型AI协同架构将传统需要2-3周的咨询项目压缩至1周，同时保证质量一致性。

**核心业务流程（四阶段）**：
1. **标准整合与聚类** - AI分析200-300页IT标准文档，生成聚类能力条目
2. **调研问卷生成** - AI设计50-100题评估问卷
3. **成熟度评分与分析** - 收集企业反馈，AI生成评估报告
4. **改进建议与对标** - AI基于标准要求生成落地措施

### 目标用户画像

**主要用户：主咨询师（李明，12年经验）**
- **痛点**：40%时间花在重复性标准分析、问卷设计
- **核心需求**：节省时间，同时保证专业质量
- **核心恐惧**："客户会发现这是AI做的，质疑我的专业性"
- **使用场景**：DevOps/ITIL/数字化转型成熟度评估项目

**次要用户：企业项目经理**
- 接收问卷、分发给团队、查看评估报告

### 核心价值主张

**对咨询师的价值**：
- 降本50%（2-3周 → 1周）
- 提质保证（三模型80%一致性验证）
- 规模化赋能（并行项目能力3-5倍提升）

**对企业客户的价值**：
- 问卷更科学（100%覆盖标准要求）
- 评估更客观（基于规范而非主观经验）
- 交付更快（1周 vs 2-3周）

### 技术独特性

**三模型众包+互审架构**：
- GPT-4 + Claude + 国产模型并行生成
- 差异点智能标注（评分差异>1.5分或关键概念不一致）
- 咨询师快速审查差异点（而非全文审查）

## Step 3: Core User Experience

### 核心用户旅程

基于user-journeys.md，提炼出主咨询师的关键体验时刻：

#### 旅程阶段1：初次接触 - 从怀疑到试探
**情境**：李明深夜整理标准文档时发现Csaas

**内心独白**：
> "写得这么专业...会不会是预设模板？我得验证一下。"

**关键行为**：
- 随机抽取10个聚类条目与原标准对照
- 发现80%准确率，但担心20%错误传递给客户
- 决定先在内部项目做对照实验

**UX设计要求**：
- 提供"查看来源标准"功能（建立信任）
- 显示置信度分数（透明化AI不确定性）
- 支持导出聚类结果（方便咨询师离线验证）

#### 旅程阶段2：真实项目中的信任危机
**情境**：李明在客户项目中发现AI严重错误（混淆"事件管理"和"问题管理"）

**内心挣扎**：
> "幸好我认真审核了。如果直接用AI的结果，客户会觉得我是个外行。"

**应对行为**：
- 手工检查全部87个条目
- 在内部笔记标注："ITIL领域，AI对流程类概念理解不足"
- 建立严格的审核机制

**UX设计要求**：
- **差异点优先审查**（重点检查AI不确定的部分）
- **领域知识提示**（"ITIL流程类概念，建议人工验证"）
- **审核checklist**（帮助咨询师建立系统化审核流程）

#### 旅程阶段3：从被动验证到主动依赖
**情境**：3个月后，李明已完成5个项目，工作模式改变

**工作流转变**：
- 以前：阅读标准文档3天 → AI综述10分钟，人工精读关键部分2小时
- 以前：手工聚类2天 → AI聚类1小时，人工调整3小时
- 以前：设计问卷1天 → AI初稿30分钟，人工优化2小时

**关键心智模型转变**：
> "AI不是替代品，而是放大器。它放大了我的生产力，但质量责任仍在我。"

**UX设计要求**：
- **效率可视化**（显示"AI为您节省了18小时"）
- **质量保障可视化**（"您审核了23个差异点，采纳了18个AI推荐"）
- **专业判断庆祝**（"✓ 已采纳您的专业判断"）

## Step 4: Desired Emotional Response

### 核心情感目标

Csaas的核心情感目标是创造**"专业自信的赋能感"（Professionally Confident Empowerment）**

这种情感由三个元素组成：

1. **赋能感（Empowerment）**
   - AI让咨询师能做以前做不到的事情（3-5倍并行项目能力）
   - "我可以服务更多客户，交付更高质量的报告"

2. **掌控感（Control）**
   - 咨询师始终是决策者，AI是协作伙伴
   - "AI提供建议，我做最终判断，这是我的专业工作"

3. **专业自信（Professional Confidence）**
   - 有AI的详细推理和三模型共识支撑，向客户展示时更有底气
   - "这不是我的直觉，这是基于127条数据和三个AI模型共识的专业分析"

### 5阶段情感旅程

#### 阶段1：首次发现 - 好奇但谨慎
**情境**：李明首次注册Csaas

**目标情感**：好奇但谨慎的兴趣

**内心独白**：
> "这AI真能帮我省70%时间？还是又一个噱头？值得一试，但我会保持警惕。"

**设计要求**：
- 产品介绍聚焦"AI协作"而非"AI自动化"
- 提供示例项目快速体验
- 首次引导强调"您始终掌控决策"

#### 阶段2：核心体验 - 效率畅快 + 专业掌控
**情境**：李明审查AI生成的三模型共识报告

**目标情感**：效率带来的畅快感 + 专业判断的掌控感

**内心独白**：
> "10秒找到差异，并排对比三个模型的逻辑，我能快速做出专业判断。这才是我想要的工具！"

**设计要求**：
- 差异点高亮醒目
- 并排展示清晰易懂
- 一键采纳/Override流畅

#### 阶段3：信任建立 - 从怀疑到依赖
**情境**：李明发现AI错误但成功纠正

**目标情感**："AI有不足，但我能兜底" 的安全感

**内心独白**：
> "AI确实会犯错，但只要我严格审核，风险可控。它放大了我的生产力。"

**设计要求**：
- 错误发现后的修正流程顺畅
- 审核checklist帮助建立系统化验证
- 显示"您的专业判断保证了质量"

#### 阶段4：习惯养成 - 高效协作的节奏感
**情境**：李明完成第5个项目，形成固定工作模式

**目标情感**：人机协作的流畅节奏感

**内心独白**：
> "AI生成初稿，我专注于20%需要深度思考的部分。这是最高效的分工。"

**设计要求**：
- 工作流自动化（上传→生成→审核→导出）
- 效率统计可视化（"本月节省54小时"）
- 快捷操作支持（键盘快捷键、批量操作）

#### 阶段5：价值实现 - 业务增长的成就感
**情境**：李明利用Csaas实现收入增长50%

**目标情感**：专业价值最大化的成就感

**内心独白**：
> "这个工具让我成为更好的咨询师。我不仅赚得更多，还能服务更多客户。"

**设计要求**：
- 业务成果dashboard（项目数、收入、客户满意度）
- 行业对标数据（"您的效率超过85%同行"）
- 推荐激励机制（邀请同行）

## Step 5: UX Pattern Analysis & Inspiration

### 核心UX模式分类

#### 模式1：差异点审查模式（创新）
**灵感来源**：GitHub Pull Request Diff View + Google Docs Suggested Edits

**适用场景**：审查三模型AI生成结果的差异点

**设计特点**：
- **并排对比**：左中右三栏展示三个模型的输出
- **差异高亮**：自动标注评分差异>1.5分或关键概念不一致
- **AI推荐标签**：显示"GPT-4和Claude一致，建议采纳"
- **快速操作**：✓采纳 / ✗拒绝 / ✎自定义

**参考产品**：
- GitHub PR Diff（并排对比清晰）
- Grammarly（AI建议 + 一键采纳）
- Figma Comments（多人协作审查）

#### 模式2：渐进式信息披露（改良）
**灵感来源**：Apple Mail (VIP优先) + Gmail Priority Inbox

**适用场景**：处理15个差异点时避免认知过载

**设计特点**：
- **三层架构**：高风险3个（默认展开） → 中风险6个（可展开） → 低风险6个（折叠）
- **优先级自动计算**：基于评分差异、领域关键性、历史错误率
- **模式切换**：⚡差异点模式 vs 📄顺序模式

**参考产品**：
- Apple Mail VIP（优先级筛选）
- Notion折叠列表（信息层次清晰）

#### 模式3：数据溯源链（创新）
**灵感来源**：Wikipedia引用系统 + Notion Backlinks

**适用场景**：咨询师向客户证明评估依据

**设计特点**：
- **每个结论可追溯**："查看推理过程" → 三模型原始输出 → IT标准原文
- **决策历史可视化**："AI建议3.5分，您调整为3.2分，理由：客户实际情况偏保守"
- **导出决策依据报告**：包含所有推理链和咨询师调整

**参考产品**：
- Wikipedia引用（严谨的溯源）
- Roam Research（双向链接）
- Notion Backlinks（关联可视化）

#### 模式4：AI推荐系统（成熟模式）
**灵感来源**：Netflix推荐 + YouTube Autoplay

**适用场景**：80%差异点可快速采纳AI推荐

**设计特点**：
- **推荐理由透明**："基于1127个历史项目，GPT-4和Claude一致性达92%"
- **置信度可视化**：高置信度（绿色徽章） / 中置信度（黄色警告） / 低置信度（红色需审查）
- **一键批量采纳**："采纳所有高置信度推荐（18个）"

**参考产品**：
- Netflix（推荐理由清晰）
- Amazon（"购买此商品的用户还购买了..."）

## Step 6: Design System Choice

### 设计系统选型决策

**选择：Ant Design 5.x + AntV数据可视化**

### 选型理由

#### 1. 企业级成熟度
- Ant Design专为企业后台设计，组件覆盖90%需求
- 大量表格、表单、数据展示组件开箱即用
- 已被阿里、腾讯、字节等大厂验证

#### 2. 数据可视化完备
- AntV (G2Plot, G6) 与Ant Design深度集成
- 支持成熟度雷达图、热力图、关系图
- 三模型对比柱状图、一致性趋势图

#### 3. 开发效率
- 完善的TypeScript支持
- 丰富的主题定制能力
- 活跃的社区和文档

#### 4. 技术栈协同
- 与Next.js完美兼容
- 支持SSR渲染优化
- Tailwind CSS可补充自定义样式

### 替代方案分析

| 设计系统 | 优势 | 劣势 | 决策 |
|---------|------|------|------|
| Ant Design | 企业级组件完备 | 体积较大 | ✅ 选择 |
| Material-UI | Google生态成熟 | 偏消费级风格 | ❌ 不适合 |
| Chakra UI | 轻量、灵活 | 组件数量少 | ❌ 需自建太多 |
| Shadcn/ui | 现代、可定制 | 无数据可视化 | ❌ 缺少关键能力 |
| 自建系统 | 完全可控 | 开发周期长 | ❌ MVP不可行 |

### 核心组件清单

**数据展示类**：
- Table（差异点列表、项目列表）
- Card（文档卡片、模型对比卡）
- Descriptions（详情展示）
- Timeline（项目进度）

**数据录入类**：
- Form（问卷设计、项目配置）
- Select（标准选择、领域筛选）
- Upload（文档上传）
- Radio/Checkbox（问卷题目）

**反馈类**：
- Modal（确认对话框）
- Message（操作成功/失败提示）
- Notification（后台任务完成通知）
- Popconfirm（删除/覆盖确认）

**导航类**：
- Menu（侧边栏导航）
- Breadcrumb（路径导航）
- Tabs（多文档切换）
- Steps（四阶段进度）

**数据可视化类（AntV）**：
- G2Plot Bar（三模型对比柱状图）
- G2Plot Radar（成熟度雷达图）
- G2Plot Heatmap（能力域热力图）
- G6 Graph（标准关联图）

## Step 7: Defining Core Experience (已整合专家反馈)

### 核心体验定义

**Csaas的核心体验是："高效审查并确认AI生成的咨询文档包"**

**关键修正（用户反馈）**：
> "核心体验要以终为始，是业务结果，也就是快速生成相对准确的聚类文档，调研问卷，问卷评分，落地优化措施。三个AI只是用于互相对比生成结果，是为了提高准确性的技术动作，而不是整个系统的核心体验。"

**核心体验不是**："一键发现三个AI在哪里产生分歧"（这只是技术手段）

**核心体验是**："快速生成相对准确的咨询文档包"（业务结果）

### 5个关键时刻

#### 1. 首次发现AI差异
**时刻描述**: 咨询师上传客户需求文档后，10秒内定位第一个高风险差异点

**用户内心独白**: "三个AI对这个能力域的评分不一致，我需要快速判断哪个更准确"

**设计要求**:
- 自动高亮前3个高风险差异点
- 并排展示三模型的推理逻辑
- 提供AI推荐（基于置信度和历史数据）

#### 2. 快速采纳推荐
**时刻描述**: 80%的差异点，咨询师能在<5秒内采纳AI推荐

**用户内心独白**: "这个推荐有理有据，我直接采纳，节省时间专注于真正有争议的20%"

**设计要求**:
- 一键采纳按钮（"✓ 采纳推荐"）
- 显示推荐理由（如"GPT-4和Claude一致，国产模型偏保守"）
- 批量采纳功能（"采纳所有低风险推荐"）

#### 3. 专业判断覆盖
**时刻描述**: 咨询师基于专业经验override AI建议时，系统"庆祝"而非质疑

**用户内心独白**: "AI不了解这个客户的特殊情况，我用我的专业判断调整"

**设计要求**:
- Override按钮明显且易用
- 确认时显示："✓ 已采纳您的专业判断"（而非"确定要覆盖AI建议吗？"）
- 记录调整理由（可选填写），便于后续追溯

#### 4. 客户沟通准备
**时刻描述**: 咨询师能快速访问完整的数据溯源链，回答客户质疑

**用户内心独白**: "客户问我为什么给他们的数字化能力打3.2分而不是3.5分，我需要展示依据"

**设计要求**:
- 每个结论都有"查看推理过程"链接
- 展示三模型的原始输出 + 数据来源
- 一键导出"决策依据报告"（包含所有AI推理和咨询师调整）

#### 5. 项目完成导出
**时刻描述**: 一键导出完整PDF，包含四阶段所有内容和决策历史

**用户内心独白**: "我要给客户发送最终报告，需要专业、完整、可追溯"

**设计要求**:
- 导出PDF包含：标准整合 + 调研报告 + 方案生成 + 差距分析
- 所有图表高清渲染
- 包含"AI辅助生成，咨询师专业审定"声明

### 成功标准（已调整为分阶段目标）

**MVP阶段目标**（首个可用版本）：

| 指标 | 目标值 | 测量方法 | 调整理由 |
|-----|--------|---------|----------|
| 项目完成时间 | <8小时 | 系统计时：上传→导出 | 原<4h不现实（PM/Architect反馈） |
| 调整率 | <40% | (修改的AI建议) / (总建议) | 原<20%对MVP过于激进 |
| 三模型一致性 | >70% | 无差异段落 / 总段落 | 原>85%需更多训练数据 |
| 成本控制 | <$5/项目 | API调用费 + 存储 | 维持不变 |
| 最大重生成次数 | ≤3次 | 系统限制 | 新增，控制成本 |

**V1.0阶段目标**（正式发布版本）：

| 指标 | 目标值 |
|-----|--------|
| 项目完成时间 | <4小时 |
| 调整率 | <20% |
| 三模型一致性 | >85% |
| 月收入增长率 | >50% |
| NPS评分 | >60 |

**北极星指标调整**（PM反馈）：
- ❌ 原指标："月处理项目数增长率>3倍"（过于关注生产力）
- ✅ 新指标："月收入增长率>30% (MVP) / >50% (V1.0)"（关注业务结果）

**调整理由**：
> "咨询师不关心自己能做多少个项目，他们关心能赚多少钱。收入增长才是真正的成功。" - John (PM)

### Novel UX Patterns（创新用户体验模式）

#### 1. Progressive Disclosure Diff Review（渐进式差异审查）

**问题**：一次性展示15个差异点会造成认知过载（Sally/UX反馈）

**解决方案**（三层渐进式披露）：

**第一层：高风险优先（默认展示）**
- 自动识别3个高风险差异点（基于评分差异>1.5分或关键能力域）
- 并排展示三模型推理
- AI推荐 + 采纳按钮

**第二层：中风险展开**
- 点击"查看6个中风险差异"
- 这部分80%可快速采纳AI推荐

**第三层：完整视图**
- 点击"查看全部15个差异点"
- 提供传统顺序审查模式

**Sally的UX优化建议**：
- 使用"3-6-15"的渐进式信息架构
- 高风险用红色标签，中风险用琥珀色
- 低风险用灰色且默认折叠

#### 2. Honest Regeneration Approach（诚实重生成策略）

**问题**：Winston指出MVP阶段无法实现真正的"智能部分重生成"（技术限制）

**解决方案**（诚实告知 + UX优化）：

**MVP阶段实现**：
- 全部重生成（重新调用三模型API）
- 但高亮变化的部分（diff算法对比新旧版本）
- 明确告知用户："为确保准确性，我们重新生成了整个章节，以下是变化的部分"

**UI设计**：
```
┌───────────────────────────────────────┐
│ ⚠️ 重新生成完成                        │
│                                       │
│ 为确保准确性，我们重新生成了"调研报告"  │
│ 全部内容。以下是与原版本的差异：        │
│                                       │
│ ✓ 3处改进（绿色高亮）                  │
│ ⚠️ 2处需确认（黄色高亮）               │
│                                       │
│ [查看详细对比] [接受全部] [逐项审查]    │
└───────────────────────────────────────┘
```

**V1.0升级计划**：
- 实现真正的部分重生成
- 依赖关系分析（>95%准确度需要更多工程投入）

#### 3. Mode Switching（审查模式切换）

**问题**：不同咨询师有不同的审查习惯

**解决方案**：提供两种审查模式

**差异点优先模式**（默认）：
- 优先展示高风险差异
- 适合经验丰富的咨询师
- 节省80%时间

**传统顺序模式**（可切换）：
- 按文档顺序逐章节审查
- 适合新手或需要深度审查的项目
- 提供完整上下文

**模式切换按钮**：
```
[⚡ 差异点模式] [📄 顺序模式]
```

### 用户心智模型（User Mental Model）

#### 心智模型1："AI是协作伙伴，不是替代者"
- 咨询师期望：AI搭建框架（80%），我专注洞察（20%）
- 设计对应：每个AI建议都保留Override入口

#### 心智模型2："我需要向客户证明专业性"
- 咨询师期望：完整的数据溯源链和推理过程
- 设计对应："查看推理过程"+"导出决策依据报告"

#### 心智模型3："效率提升不能以质量为代价"
- 咨询师期望：快速但准确
- 设计对应：三模型共识机制 + 差异点审查

#### 心智模型4："我能控制成本"
- 咨询师期望：知道每次重生成的成本
- 设计对应：实时显示API费用，限制重生成次数（≤3次）

#### 心智模型5："客户会发现这是AI做的"（核心恐惧）

**咨询师的核心恐惧**（John/PM反馈）：
> "咨询师最怕客户质疑：'你这不就是AI生成的吗？我为什么要付高价？'"

**消解策略**：

**1. 客户沟通话术库**
提供预设的客户沟通脚本：

**话术1（强调专业判断）**：
> "我们使用AI工具快速搭建了行业标准框架，但所有关键判断都基于我们对贵公司的深入理解和20年行业经验。AI节省了80%的框架搭建时间，让我们能投入更多精力在20%的定制化洞察上。"

**话术2（强调质量保障）**：
> "我们采用三个顶级AI模型交叉验证，确保没有遗漏关键点。但最终的优先级排序和方案选择，都是基于我们对贵公司战略的理解和行业最佳实践。"

**话术3（强调数据支撑）**：
> "这份报告的每个结论都有完整的数据溯源。如果您对任何评分有疑问，我可以展示支撑这个结论的127条数据点和三个AI模型的一致性分析。"

**2. 价值重新定位**
- 在系统中强化："AI辅助生成，咨询师专业审定"
- 导出报告中注明："本报告由[咨询师姓名]基于AI辅助分析和专业判断完成"

**3. 专业权威建立**
- 咨询师的调整被系统"庆祝"
- 最终报告中突出显示"咨询师专业判断"的部分

## Step 8: Visual Design Foundation (已整合专家反馈)

### 核心设计决策摘要

**颜色系统**：
- 主色调：#1E3A8A（深蓝色，传达专业可信赖）
- 成功色拆分：#10B981（背景/边框）+ #059669（文字，WCAG AA 4.69:1对比度）
- 警告色：#F59E0B（中等风险差异点）
- 错误色：#EF4444（高风险差异点）

**排版系统**：
- 字体：Inter（英文）+ 系统字体（PingFang SC / Microsoft YaHei）
- 基准字号：14px，行高1.71（24px）优化中文阅读
- 字重：400正文、600标题、700关键指标

**间距与布局**：
- Card padding：20px（默认）/ 24px（大卡片）
- 24列栅格系统，响应式992px断点
- 平衡密度，适合长时间文档审查

**技术栈**：
- Next.js 14 + Ant Design 5.x + Tailwind CSS
- 字体通过next/font优化加载
- Tailwind配置：preflight: false（避免与Ant Design冲突）

**Ant Design主题配置（核心代码）**：
