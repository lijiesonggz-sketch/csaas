# Csaas MVP规格说明书

**版本**: v1.0
**日期**: 2025-12-25
**阶段**: DEMO阶段（0-6个月）
**目标**: 验证三模型AI协同架构的技术可行性和市场价值

---

## 执行摘要

**MVP核心假设**：三模型AI协同架构能够产出质量接近人工咨询师、但效率提升10倍的IT咨询成熟度评估报告。

**验证方式**：完成10-20个真实咨询项目，AI审核通过率 > 80%，报告覆盖率 > 95%，准确性达到人工的90%+。

**时间线**：0-6个月（建议4个月完成核心功能，2个月真实项目验证）

**成功标准**：与1-2家中小咨询公司建立深度合作，他们愿意用真实客户项目测试系统。

## 一、MVP必须实现的功能（P0优先级）

### 1.1 核心AI流程（5个环节）

**FR7 - 标准文档综述生成** [P0]
- **能力**：主咨询师可以触发AI对标准文档进行综述生成（2-3页摘要）
- **三模型并行**：GPT-4、Claude、国产模型各生成1份综述
- **质量验证**：分层相似度验证（语义等价性≥80%）
- **输出**：投票选最佳综述 + 一致性评分

**FR8 - 智能聚类** [P0]
- **能力**：主咨询师可以触发AI对标准条款进行智能聚类（生成12-15个类别）
- **三模型并行**：各生成1份聚类方案
- **质量验证**：
  - 结构一致性≥90%（类别划分逻辑）
  - 语义等价性≥80%
  - **覆盖率检查**（FR17）：自动检测是否覆盖所有条款，列出遗漏条款
- **输出**：投票选最佳聚类 + 覆盖率报告

**FR9 - 成熟度矩阵生成** [P0]
- **能力**：主咨询师可以触发AI生成成熟度矩阵（5级成熟度 × 聚类维度）
- **三模型并行**：各生成1份矩阵
- **质量验证**：结构一致性验证（矩阵维度、单元格填充完整性）
- **输出**：投票选最佳矩阵

**FR10 - 问卷生成** [P0]
- **能力**：主咨询师可以触发AI生成问卷题目（50-100题，含题目、选项、评分规则）
- **三模型并行**：各生成1份问卷
- **质量验证**：
  - 语义相似度验证
  - **问卷覆盖完整性**（FR21）：验证问卷对标准维度的覆盖
- **输出**：投票选最佳问卷

**FR11 - 落地措施生成** [P0]
- **能力**：主咨询师可以触发AI生成落地措施（基于问卷结果的差距分析+改进建议）
- **三模型并行**：各生成1份落地措施清单
- **质量验证**：语义相似度验证
- **输出**：投票选最佳措施清单

### 1.2 三模型并行调用架构

**FR14 - 实时进度显示** [P0]
- 显示"模型1已完成（3分12秒），模型2进行中（已用2分钟，预计还需3分钟），模型3排队中"
- 每30秒更新一次进度

**FR83-FR84 - 异步执行与通知** [P0]
- 任务提交后在后台异步执行，用户可继续其他工作
- 完成后通过邮件通知（通知成功率≥95%）

**FR76-FR78 - 多级降级策略** [P0]
- **Level 1（理想）**：三模型全部成功 → 正常质量验证流程
- **Level 2（可接受）**：两个模型成功，一个失败 → 基于两个模型验证（降低一致性要求到70%）
- **Level 3（降级）**：仅一个模型成功 → 标记为"低置信度"，强制人工深度审核
- **Level 4（兜底）**：三个模型全失败 → 提供"手工模式"，AI辅助检索但不自动生成

**FR77 - 单模型失败继续运行** [P0]
- 单个模型失败不影响其他模型，失败时自动重试（最多3次，间隔5-10分钟）

**FR76 - API超时自动切换** [P0]
- 单个模型API超时时自动切换到备用API密钥（切换时间≤10秒）

### 1.3 质量验证能力

**FR12-FR13 - 相似度计算与质量判定** [P0]
- **分层验证**：
  - Level 1: 结构一致性≥90%（聚类类别、矩阵维度）
  - Level 2: 语义等价性≥80%（使用OpenAI embedding计算余弦相似度）
  - Level 3: 细节一致性≥60%（措辞、排序）
- **质量等级**：
  - 高置信度（≥85%）：80%的AI生成结果应达到此级别
  - 中等置信度（75-85%）
  - 低置信度（<75%）：强制人工深度审核

**FR17-FR18 - 覆盖率检查** [P0]
- 自动检测AI聚类是否覆盖标准文档的所有条款
- 精确列出遗漏条款（如"条款3.2.1、条款5.1.3未被聚类"）
- 覆盖率目标：≥95%（人工平均98%）

**FR19-FR20 - 高风险条款识别** [P0]
- 自动识别涉及安全、合规、法律责任的条款
- 标记三模型生成结果的分歧点
- **关键词匹配**：安全、合规、法律责任、数据保护、访问控制
- **标准章节匹配**：ISO 27001第5-8章、COBIT第3章

**FR21 - 问卷覆盖完整性验证** [P0]
- 验证问卷对标准维度的覆盖完整性
- 确保所有聚类类别都有对应问题

---

### 1.4 人工审核能力

**FR24 - 原文溯源** [P0]
- 查看AI生成结果与原始标准文档的对照
- 每个聚类/矩阵单元格可追溯到标准文档原文

**FR25 - AI逻辑说明** [P0]
- 查看AI的聚类逻辑说明（为什么将条款A和B聚在一起）
- AI必须提供可追溯的推理过程

**FR26 - 一致性评分查看** [P0]
- 查看三模型生成结果的一致性评分（结构层、语义层、细节层）
- 可视化展示分歧点

**FR27-FR29 - 审核操作** [P0]
- **接受**：接受AI生成的结果并进入下一环节
- **修改**：修改AI生成的部分内容（如调整聚类、修改问卷题目）
- **拒绝**：拒绝AI结果并要求重新生成

**FR30 - 高风险条款强制审核** [P0]
- 即使三模型一致性100%，高风险条款仍需主咨询师逐条确认
- 系统提供"挑战性问题"引导审核
- 咨询师必须勾选"我已仔细核对原文，确认无误"才能继续

**渐进式披露三阶段**（设计要求）：
1. **独立思考阶段** - 先展示原始标准文档，激活咨询师独立判断
2. **对比发现阶段** - 展示AI结果+一致度+逻辑说明，高亮高风险项
3. **深度验证阶段** - 针对高风险项，展示原文对照+挑战性问题引导

### 1.5 项目管理能力

**FR1-FR3 - 项目基础操作** [P0]
- **创建项目**：指定项目名称、客户信息、咨询领域和IT标准
- **上传标准文档**：PDF/Word格式，最大200页
- **查看进度**：查看整体进度（已完成环节、进行中环节、待启动环节）

**FR50 - 数据存储** [P0]
- 存储项目的所有AI生成内容和人工审核记录
- 支持版本管理（咨询师修改后保留AI原始版本）

**FR51 - 数据标记** [P0]
- 将项目数据标记为独占（仅咨询公司可访问）或共享（参与匿名对标）
- **默认独占**：DEMO阶段所有数据默认独占，不参与对标

### 1.6 问卷管理能力

**FR34-FR36 - 问卷定制与分发** [P0]
- **自定义问卷**：新增、删除、修改措辞
- **指定填写角色**：为问卷指定填写角色（如技术负责人、运维负责人、管理层）
- **邮件分发**：发送问卷邀请给企业方指定人员

**FR37-FR39 - 问卷填写** [P0]
- 被调研者可以在线填写问卷
- **自动保存草稿**（NFR-U6）：每60秒自动保存，用户手动操作时立即保存
- 被调研者可以查看自己已提交的问卷答案

**FR78-FR79 - 填写体验优化** [P0]
- 显示填写进度条（已完成X/总共Y题）
- 随时保存并退出（自动保存草稿）

---

### 1.7 报告生成能力

**FR44-FR45 - 基础报告导出** [P0]
- 导出差距分析报告（PDF格式，含成熟度评分、低分项说明、改进建议）
- 包含成熟度雷达图（可视化展示各维度得分）

**MVP阶段报告生成方式**：
- ❌ 不做自动生成与导出（手工整理）
- 主咨询师手工整理AI生成内容 + 问卷结果，制作报告
- 系统提供所有必要数据（成熟度评分、落地措施、问卷统计）

---

### 1.8 权限与访问控制

**FR57-FR58 - 基础权限模型** [P0]
- 区分主咨询师、企业PM、被调研者三种角色的访问权限
- 主咨询师可以邀请企业PM加入项目并授予查看权限

**MVP阶段简化**：
- ❌ 不做多租户权限管理（DEMO阶段单租户即可）
- 仅实现单个咨询公司的项目管理
- 不做平台管理员功能

---

### 1.9 异常处理与容错

**FR79 - 断点续传** [P0]
- 长文档分段处理，部分失败时只重试失败的部分，不影响已成功的部分

**FR78 - 手工模式兜底** [P0]
- 三个模型全部失败时提供"手工模式"（AI辅助检索但不自动生成）
- 系统展示标准文档，咨询师可手工创建聚类/矩阵/问卷

## 三、MVP成功标准

### 3.1 技术可行性验证

**AI质量指标**：
- **AI审核通过率** > 80%（即80%的AI生成内容无需大幅修改<20%内容）
- **报告覆盖率** > 95%（AI聚类结果覆盖标准文档所有条款）
- **三模型一致性** ≥ 75%（分层相似度验证通过率）

**系统可靠性指标**：
- **任务完成率**（非降级模式）≥ 90%
- **系统可用性**（uptime）≥ 99.5%（每月计划外停机≤3.6小时）
- **异步任务通知成功率** ≥ 95%

**性能指标**：
- **AI任务执行时间** P50 ≤ 25分钟，P95 ≤ 35分钟（三模型并行处理完成）
- **审核界面响应时间** P95 ≤ 2秒
- **并发任务处理能力** 支持10个并发任务

### 3.2 质量对照实验

**实验设计**：
- **样本**：3个真实IT标准文档（ISO 27001附录A、等保2.0三级、COBIT 2019某管理域）
- **对照组**：2个资深IT咨询师（5-10年经验）手工聚类 vs Csaas三模型系统自动聚类
- **评估维度**：
  - 覆盖率：系统 ≥ 95%（人工平均98%）
  - 准确性：系统平均分 ≥ 人工平均分的90%（2个独立专家盲评打分1-10分）
  - 聚类逻辑合理性：系统 ≥ 人工的85%
  - 效率：系统总时间 ≤ 人工的20%

**实验时间**：2周（与DEMO核心功能开发并行）

**实验成本**：约1.3万元
- 2个资深咨询师：9,000元（2人×3个标准×5小时×300元/小时）
- 2个独立专家评审：4,000元（2人×2,000元）
- API调用成本：约300-500元

**输出交付**：
1. 《Csaas质量对照实验报告》
2. 实验数据仪表板（可视化对比）
3. Executive Summary更新（将验证数据写入PRD）

### 3.3 真实项目验证

**项目数量**：10-20个真实咨询项目

**执行方式**：
- 创始人作为主咨询师亲自验证每个项目
- 与1-2家中小咨询公司建立深度合作关系（可考虑股权绑定）
- 用真实客户项目测试系统（客户需明确知情并同意）

**验证指标**：
- **项目完成周期**：从2-3周缩短至 ≤ 1周
- **审核总时间**：5个AI生成环节总审核时间 ≤ 3小时
- **客户满意度**：企业客户对评估报告的满意度评分 ≥ 4.5/5
- **咨询师满意度**：主咨询师对系统的满意度评分 ≥ 4/5

### 3.4 商业验证

**深度合作**：
- 与1-2家中小咨询公司建立深度合作关系
- 咨询公司愿意用真实客户项目持续测试系统
- 咨询公司创始人认可产品价值，愿意付费或股权合作

**用户反馈**：
- 咨询师认为"值了"的关键时刻：
  1. 第一次看到AI生成的聚类结果，覆盖率达95%+，逻辑清晰
  2. 完成审核只用了2-3小时，而不是以往2-3周的重复劳动
  3. 客户对报告质量非常满意，评分4.5+
  4. 能够同时服务6个客户项目，收入和影响力翻倍

## 四、非功能需求（NFR）- P0级别

### 4.1 AI治理与质量

**NFR-AI1: AI共识一致性** [P0]
- 三模型并行生成的结果达到共识的比例 ≥ 80%（DEMO阶段可接受80%，Growth阶段需≥85%）
- 测量：(通过相似度验证的任务数 / 总任务数) × 100%

**NFR-AI2: 单模型响应时间SLA** [P0]
- 单个AI模型处理标准文档（200-300页）的响应时间 P95 ≤ 15分钟
- 降级策略：单模型超时15分钟后，继续等待其他2个模型（最多30分钟总超时）

**NFR-AI5: AI共识超时处理** [P0]
- 三模型分歧持续超过3次重试（总计30分钟）时，系统自动降级为"低置信度"模式

**NFR-AI6: 部分共识接受标准** [P0]
- 当仅2个模型达成一致（相似度≥90%），第3个模型失败或分歧时，系统自动接受2模型共识并标记为"中等置信度"

### 4.2 性能

**NFR-P1: AI任务执行时间** [P0]
- 三模型并行处理完成时间 P50 ≤ 25分钟，P95 ≤ 35分钟
- 业务影响：慢速AI会毁掉demo转化率

**NFR-P2: 审核界面响应时间** [P0]
- 用户交互操作（展开聚类详情、查看原文对照、切换模型结果）响应时间 P95 ≤ 2秒

**NFR-P4: 并发任务处理能力** [P0]
- 支持10个并发AI生成任务，响应时间 < 3秒

**NFR-P5: 问卷填写加载性能** [P0]
- 问卷页面Time to Interactive (TTI) ≤ 3秒（含50-100题的完整问卷）

**NFR-P6: 进度更新频率** [P0]
- AI任务进度每30秒更新一次，显示当前完成百分比、预计剩余时间、当前执行步骤描述

### 4.3 可靠性

**NFR-R1: 系统可用性** [P0]
- 正常运行时间(uptime) ≥ 99.5%（每月计划外停机 ≤ 3.6小时）

**NFR-R2: AI任务完成率** [P0]
- 三模型并行调用任务完成率（非降级模式）≥ 90%

**NFR-R3: 数据持久性** [P0]
- 所有项目数据、AI生成内容、人工审核记录零丢失
- 每日全量备份 + 每小时增量备份，保留90天

**NFR-R4: 降级模式可用性** [P0]
- 降级模式（Level 2/3/4）必须可靠执行核心能力
- 混沌工程测试各级降级场景，功能可用性 100%

**NFR-R5: 异步任务通知可靠性** [P0]
- 任务完成后邮件通知发送成功率 ≥ 95%

### 4.4 安全性

**NFR-S1: 数据传输加密** [P0]
- 所有网络传输强制使用TLS 1.2或更高版本
- SSL Labs评分 ≥ A

**NFR-S2: 敏感数据存储加密** [P0]
- 独占数据（企业名称、问卷回复、报告内容）使用AES-256加密存储

**NFR-S3: 身份认证安全** [P0]
- 密码强度：≥8字符，包含大小写字母+数字+特殊符号

## 五、技术架构决策（ADR摘要）

### ADR #1: 多租户架构选择
- **决策**：采用共享数据库+tenant_id隔离模式
- **MVP实现**：单租户，但架构预留扩展性

### ADR #2: 三模型并行调用架构
- **决策**：异步队列 + 多级降级
- **实现**：
  - 用户提交任务 → Task Manager创建任务记录
  - 三个Worker并行调用API（独立执行，互不等待）
  - Aggregator监听：三个结果都完成时触发
  - 相似度计算 → 投票选最佳 → 通知用户

### ADR #3: 数据隔离实施细节
- **决策**：纵深防御4层
  1. 应用层：所有查询强制WHERE tenant_id = current_tenant
  2. 数据库层：Row Level Security (RLS)
  3. 网络层：租户数据物理隔离（Growth阶段）
  4. 审计层：所有跨租户访问尝试记录

### ADR #4: AI生成过程的事件溯源架构
- **决策**：完整记录AI生成过程（事件溯源）
- **存储内容**：
  - 三模型的原始输出
  - 相似度计算详情
  - 投票选择逻辑
  - 人工审核记录（接受/修改/拒绝）
  - 版本历史（AI版本 vs 咨询师调整版本）

## 八、风险缓解策略

### 8.1 技术风险

**风险1：AI模型质量不达标**
- **缓解**：质量对照实验提前验证（2周内完成）
- **Plan B**：如果准确性<90%，调整提示词或切换模型

**风险2：三模型API成本失控**
- **缓解**：成本监控告警（单项目>30元告警）
- **Plan B**：使用更便宜的模型（GPT-4 → GPT-4o-mini）

**风险3：API稳定性问题**
- **缓解**：多级降级策略 + 备用API密钥
- **Plan B**：手工模式兜底（AI辅助检索但不自动生成）

### 8.2 业务风险

**风险4：咨询师抵触AI工具**
- **缓解**：渐进式披露设计 + "赋能而非替代"定位
- **Plan B**：调整营销话术，强调"AI是助手，咨询师是专家"

**风险5：合作咨询公司流失**
- **缓解**：股权绑定 + 深度服务支持
- **Plan B**：快速迭代优化产品体验

### 8.3 质量风险

**风险6：AI错误导致客户索赔**
- **缓解**：三重防线验证 + 高风险条款强制审核
- **Plan B**：质量保证基金（预留50万/项目赔付上限）

**风险7：数据泄露**
- **缓解**：TLS加密 + 数据库加密 + 审计日志
- **Plan B**：购买网络安全保险
