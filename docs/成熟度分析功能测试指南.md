# 成熟度分析功能测试指南

## 📋 测试前准备

### 1. 确认服务运行状态

#### 检查后端服务 (端口 3000)
```bash
# Windows CMD
netstat -ano | findstr ":3000"
```

**期望结果**: 应该看到 `LISTENING` 状态

#### 检查前端服务 (端口 3001)
```bash
# Windows CMD
netstat -ano | findstr ":3001"
```

**期望结果**: 应该看到 `LISTENING` 状态

### 2. 如果服务未运行，启动服务

#### 启动后端 (在 `backend` 目录)
```bash
cd D:\csaas\backend
npm run start:dev
```

#### 启动前端 (在 `frontend` 目录，新开终端)
```bash
cd D:\csaas\frontend
npm run dev
```

### 步骤 3: 快速填写问卷 (测试模式)

为了快速测试，您可以:

#### 方法A: 手动随机选择 (推荐，更真实)
- 每道题随机选择1-2个选项
- 可以混合选择不同成熟度等级
- 大约需要5-10分钟

#### 方法B: 使用Excel导入功能 (最快)
1. 点击"导出为Excel"按钮
2. 打开下载的Excel文件
3. 在"您的选择"列随机填写:
   - 单选题填写: `A` 或 `B` 或 `C`
   - 多选题填写: `A,B` 或 `B,C,D` (用逗号分隔)
4. 保存Excel文件
5. 点击"导入Excel"按钮，选择刚才编辑的文件
6. 系统自动填充所有答案

**期望看到** (导入后):
- 所有题目的答案已自动填充
- 进度条显示100%
- 可以看到总分

---

### 步骤 4: 提交问卷

1. 滚动到页面底部

**期望看到**:

### 步骤 5: 验证成熟度分析页面

页面URL应该类似:
```
http://localhost:3001/survey/analysis?surveyId=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

#### 5.1 检查页面标题区域
**应该显示**:
- ✅ 页面标题: "成熟度分析报告"
- ✅ 受访者信息: 姓名、部门、职位
- ✅ 提交时间

#### 5.2 检查冲突警告 (如果有)
**如果存在冲突，应该看到**:
- ✅ 醒目的警告框 (黄色或红色背景)
- ✅ 冲突数量显示
- ✅ 严重程度标识 (LOW/MEDIUM/HIGH)
- ✅ 冲突详情列表:
  - 聚类内冲突: 显示聚类名称、方差值、建议
  - 聚类间冲突: 显示前置/依赖关系、建议

**测试点**: 如果聚类内某5个问题得分差异很大(如1分、5分、2分、4分、1分)，应该触发聚类内冲突

#### 5.3 检查总体成熟度卡片
**应该显示**:
- ✅ 大号数字显示成熟度等级 (如 `2.89`)
- ✅ 等级标签 Badge (如 "初步规范级")
- ✅ 说明文字
- ✅ 计算公式 (格式: `318 / 550 × 5 = 2.89`)
- ✅ 总得分和满分

**验证**:
- 手动计算: 总得分 ÷ 满分 × 5 = 成熟度
- 检查等级是否正确:
  - 1.0-2.0: 初始级
  - 2.0-3.0: 初步规范级
  - 3.0-4.0: 充分规范级
  - 4.0-4.5: 系统优化级
  - 4.5-5.0: 卓越级

#### 5.4 检查成熟度分布
**应该显示**:
- ✅ 5个等级柱状图 (Level 1 到 Level 5)
- ✅ 每个等级显示聚类数量 (可能是小数，因为按比例分配)
- ✅ 总和应约等于22 (总聚类数)

#### 5.5 检查 TOP 5 短板和优势
**左侧 - TOP 5 短板 (红色)**:
- ✅ 显示5个成熟度最低的聚类
- ✅ 每个显示: 排名、聚类名称、成熟度、差距值
- ✅ 按成熟度从低到高排序

**右侧 - TOP 5 优势 (绿色)**:
- ✅ 显示5个成熟度最高的聚类
- ✅ 每个显示: 排名、聚类名称、成熟度、优势值
- ✅ 按成熟度从高到低排序

#### 5.6 检查统计信息
**应该显示**:
- ✅ 总问题数: 110
- ✅ 总聚类数: 22
- ✅ 平均聚类成熟度: 应该接近总体成熟度
- ✅ 成熟度标准差: 数值合理 (通常0.3-0.8)

#### 5.7 检查各聚类详情 (最重要)
**应该显示**:
- ✅ 22个聚类卡片
- ✅ 每个聚类显示:
  - 聚类名称
  - 成熟度等级 Badge
  - 是否为短板 (红色"短板"标签)
  - 维度分类
  - 问题数量 (应该是5)
  - 计算公式

**交互测试**:
1. 点击任意一个聚类卡片
2. **期望**: 卡片展开，显示该聚类的5个问题详情
3. **应该看到**:
   - 每个问题的完整文本
   - 每个问题的得分 (x/5)
   - 每个问题的等级 (Level x)
4. 再次点击同一聚类

**建议测试的聚类**:
- 测试短板聚类 (成熟度最低的)
- 测试优势聚类 (成熟度最高的)

## 🔍 重点测试场景

### 场景 1: 测试冲突检测

**目标**: 触发聚类内冲突

**操作**:
1. 在Excel中，找到同一个聚类的5个问题
2. 故意填写差异很大的答案，例如:
   - 问题1: 选择A (得分1)
   - 问题2: 选择E (得分5)
   - 问题3: 选择A (得分1)
   - 问题4: 选择E (得分5)
   - 问题5: 选择C (得分3)
3. 提交后查看分析页面

**期望结果**:
- ✅ 页面顶部显示冲突警告
- ✅ 该聚类被标记为有冲突
- ✅ 显示方差值 (应该 > 2.0)
- ✅ 显示建议: "请检查是否存在理解偏差或填写错误"

---

### 场景 2: 测试不同成熟度等级

**测试低成熟度** (目标: 1.0-2.0):
- 大部分问题选择A或B选项 (低分)

**测试中等成熟度** (目标: 2.5-3.5):
- 大部分问题选择B、C、D选项

**测试高成熟度** (目标: 4.0-5.0):
- 大部分问题选择D或E选项 (高分)

**验证**:
- 总体成熟度数值正确
- 等级标签正确
- 说明文字匹配等级

---

### 场景 3: 测试聚类详情展开

**操作**:
1. 找到标记为"短板"的聚类
2. 点击展开
3. 查看5个问题的得分

**验证**:
- ✅ 5个问题的得分总和 = 聚类总分
- ✅ 计算公式正确: `总分 / 25 × 5 = 成熟度`
- ✅ 如果有冲突，5个问题得分应该差异较大

## ❌ 常见问题排查

### 问题 1: 提交后没有跳转到分析页面

**可能原因**:
- 浏览器控制台有JavaScript错误
- 后端API调用失败

**排查步骤**:
1. 按F12打开浏览器开发者工具
2. 查看Console标签页是否有红色错误
3. 查看Network标签页，找到 `submit` 和 `analyze` 请求
4. 检查返回状态码是否为200

---

### 问题 2: 分析页面显示"Survey must be submitted before analysis"

**原因**: 问卷状态不是 `submitted`

**解决**:
- 确保点击了"提交问卷"按钮
- 检查后端数据库survey_responses表，status字段应该是'submitted'

---

### 问题 3: 页面一直显示Loading

**排查**:
1. 检查浏览器控制台Network标签
2. 找到 `analyze` 请求
3. 查看是否返回了数据
4. 如果请求失败，检查surveyId是否正确

---

### 问题 4: 冲突检测没有显示

**这是正常的！**
- 冲突检测需要满足特定条件:
  - 聚类内方差 > 2.0 才会触发
  - 随机填写可能不会触发冲突
- 如果想测试冲突，参考"场景1: 测试冲突检测"
