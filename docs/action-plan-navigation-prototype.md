# 改进措施页面 - 聚类导航优化方案原型图

## 一、当前问题分析

### 问题 1: 框太小，文字溢出
```
┌─────────────────────┐
│ 1. 人工智能与算法  │ ← 文字突破边框
│    2.3 → 3.5 (差距  │
│    1.2) [12条措施]  │
└─────────────────────┘
```

### 问题 2: 点击跳转失效
- 点击按钮无法定位到对应措施
- 措施卡片没有 ID 或标记

### 问题 3: 信息扁平化
- 16个聚类全部平铺显示
- 缺少层级关系（一层分类 → 二层聚类）
- 信息密度过高，难以快速定位

## 二、优化方案设计

### 数据结构
```
4 个一层分类 (Categories)
├── 1. 安全治理与组织建设 (3 个聚类)

├── 3. 技术保护与系统安全 (5 个聚类)
└── 4. 风险监测、审计与应急响应 (3 个聚类)

总计 16 个二层聚类 (Clusters)
```

## 五、视觉设计规范

### 一层分类卡片
- **尺寸**: 全宽（100%）
- **内边距**: 20px
- **边框**: 1px solid #e5e7eb
- **圆角**: 12px
- **背景**: 白色
- **悬停**: 背景色 #f9fafb，边框色 #d1d5db

### 二层聚类卡片
- **尺寸**: 宽度 33%（一行显示3个）或 50%（一行显示2个）
- **最小宽度**: 320px
- **内边距**: 16px
- **边框**: 2px solid transparent
- **圆角**: 8px
- **背景**: 渐变 #f9fafb → #ffffff
- **悬停**: 边框色 #3b82f6，阴影 0 4px 12px rgba(59,130,246,0.15)
- **可点击区域**: 整个卡片（光标 pointer）

### 措施详情卡片
- **边框**: 左边框 4px solid（优先级颜色）

  - 低优先级: #3b82f6（蓝色）
- **高亮状态**: 背景色 #fef3c7（黄色高亮）

## 六、数据流设计

### 1. 获取层级数据
```typescript
// 从聚类任务结果中提取层级关系
const clusteringResult = await getClusteringTaskResult(projectId)
const categories = clusteringResult.content.categories // 4个一层分类

// 建立映射
const categoryMap = new Map()
categories.forEach(cat => {
  cat.clusters.forEach(cluster => {
    categoryMap.set(cluster.name, cat.name) // 二层聚类名 → 一层分类名
  })
})
```

### 2. 计算聚类统计
```typescript
// 为每个聚类计算统计数据
const clusterStats = detailedMeasures.reduce((acc, measure) => {
  const clusterName = measure.clusterName
  if (!acc[clusterName]) {
    acc[clusterName] = {
      measures: [],
      currentLevels: [],
      targetLevels: []
    }
  }
  acc[clusterName].measures.push(measure)
  acc[clusterName].currentLevels.push(measure.currentLevel)
  acc[clusterName].targetLevels.push(measure.targetLevel)
  return acc
}, {})

// 计算平均值
Object.entries(clusterStats).forEach(([name, stats]) => {
  const avgCurrent = stats.currentLevels.reduce((a,b) => a+b) / stats.currentLevels.length
  const avgTarget = stats.targetLevels.reduce((a,b) => a+b) / stats.targetLevels.length
  // 存储统计数据
})
```

### 3. 分类级别统计
```typescript
// 为每个一层分类计算统计数据
const categoryStats = categories.map(cat => {
  const clusterNames = cat.clusters.map(c => c.name)
  const categoryMeasures = detailedMeasures.filter(m =>
    clusterNames.includes(m.clusterName)
  )

  return {
    name: cat.name,
    description: cat.description,
    clusterCount: cat.clusters.length,

  }
```

## 七、功能要点

### ✅ 优点
1. **层次清晰**: 两级导航，从概括到具体
2. **渐进式展示**: 默认折叠，按需展开，减少信息过载
3. **视觉友好**: 卡片尺寸合理，不会文字溢出
4. **操作直观**: 点击展开/收起，箭头图标明确
5. **统计完整**: 每层级都显示成熟度和措施数量
6. **快速定位**: 点击"查看措施"直接滚动到对应位置
7. **可访问性**: 支持 URL hash（#cluster-1-2），可直接分享链接

### 📊 可选增强功能
1. **搜索框**: 在聚类名称中搜索关键词
2. **筛选器**: 按优先级筛选（只显示高优先级措施）
3. **排序**: 按差距大小、措施数量排序
4. **进度追踪**: 显示每个聚类的完成进度
5. **折叠全部/展开全部**: 快速操作按钮

## 十、响应式设计

- 二层聚类: 2列（每列 50%）

- 一层分类: 1列（全宽）

## 总结

这个优化方案将现有的扁平化 16 个聚类导航，重构为 **两层递进式导航**：

1. **第一层**: 4 个一层分类（概括视图）
2. **第二层**: 点击展开显示该分类下的二层聚类（详细视图）
3. **第三层**: 点击"查看措施"滚动到具体措施（执行视图）

**核心优势**:
- ✅ 解决框太小、文字溢出问题
- ✅ 提供清晰的层级关系
- ✅ 渐进式展示，降低认知负担
- ✅ 保持所有统计信息完整
- ✅ 支持快速定位和跳转
- ✅ 响应式设计，适配各种设备
