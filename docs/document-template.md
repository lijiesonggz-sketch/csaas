## 格式要求

### ✅ 推荐格式

1. **章节标题**：
   - 使用"第X章"格式
   - 例如：第一章 总则

2. **条款编号**：
   - 使用"第X条"格式（中文数字）
   - 例如：第五条、第十条、第二十三条

3. **条款内容**：
   - 每个条款独占一段或数段
   - 以"第X条"开头
   - 内容清晰完整

4. **空行分隔**：
   - 章节之间空一行
   - 条款之间可以空一行（可选）

### ❌ 避免的格式

1. **圆点项目符号**：
   ```
   ❌ • 总  则
   ❌ • 业务数据分类分级
   ```

2. **纯数字编号**：
   ```
   ❌ 1. 为规范...
   ❌ 2. 本办法所称...
   ```

3. **无编号段落**：
   ```
   ❌ 为规范银行业保险业...
   ❌ 本办法所称银行保险机构...
   ```

4. **混合格式**：
   ```
   ❌ 第一章
   ❌ • 1.1 要求
   ❌ • 1.2 职责
   ```

## 格式转换示例

### 示例1：圆点转标准

**转换前**：
```
• 总  则
• 为规范银行业保险业数据处理活动...
• 本办法所称银行保险机构...
```

**转换后**：
```
第一章 总则

第一条 为规范银行业保险业数据处理活动...

第二条 本办法所称银行保险机构...
```

### 示例2：数字编号转标准

**转换前**：
```
1. 为规范银行业保险业数据处理活动...
2. 本办法所称银行保险机构...
```

**转换后**：
```
第一条 为规范银行业保险业数据处理活动...

第二条 本办法所称银行保险机构...
```

## 特殊情况处理

### 1. 子条款格式

如果条款包含子项，建议格式：
```
第十条 银行保险机构应当建立数据安全责任制，包括：

（一）明确第一责任人和直接责任人；
（二）建立数据安全管理制度；
（三）开展数据安全培训。
```

### 2. 表格处理

如果文档包含表格，建议转换为文字描述：
```
原表格：
| 数据级别 | 定义       |
| 核心数据 | 高覆盖度   |

转换为：
第十一条 根据数据的重要性和敏感程度，将数据分为核心数据、重要数据、一般数据。
核心数据是指对领域、群体、区域具有较高覆盖度的数据。
```

### 3. 附件处理

如果文档有附件，建议将附件内容直接嵌入正文，或单独上传后再合并。
