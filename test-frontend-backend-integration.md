# Frontend-Backend Integration Test Guide

## Phase 2 Week 5 - 综述生成功能测试

### 测试环境准备

1. **后端运行检查**
   ```bash
   # 检查后端是否在运行
   curl http://localhost:3000/health

   # 如果未运行，启动后端
   cd backend
   npm run start:dev
   ```

2. **前端运行检查**
   ```bash
   # 启动前端开发服务器
   cd frontend
   npm run dev
   ```

3. **Redis检查**
   ```bash
   # 确保Redis正在运行（BullMQ依赖）
   # Windows: redis-server
   # Docker: docker run -d -p 6379:6379 redis
   ```

4. **数据库检查**
   ```bash
   # 确保PostgreSQL正在运行
   # 确保迁移已执行
   cd backend
   npm run migration:run
   ```

- 点击"开始生成综述"按钮

#### 4. 监控实时进度
观察以下元素：

- 三个模型状态框应依次变化：
  - GPT-4: 等待中 → 处理中 → 已完成
  - Claude: 等待中 → 处理中 → 已完成

#### 5. 查看生成结果
完成后应显示：

**生成信息卡片**
- 任务ID
- 生成时间
- 选中模型（GPT-4/Claude/通义千问）
- 置信度（HIGH/MEDIUM/LOW）
- 审核状态（待审核）
- 版本号

**质量评分卡片**
- 结构一致性：应≥90%（绿色进度条）
- 语义一致性：应≥80%（绿色进度条）
- 细节一致性：应≥60%（绿色进度条）

**一致性报告卡片**
- 一致点列表（可折叠）
- 差异点列表（如有）
- 高风险差异列表（如有）

**综述内容卡片**
- 标题
- 概述（200-300字）
- 关键领域（每个显示重要性标签：高/中/低）
- 适用范围
- 关键要求（列表形式）
- 合规级别说明

### 常见问题排查

#### 问题1: WebSocket连接失败
- 检查 `frontend/.env.local` 中 `NEXT_PUBLIC_API_URL` 是否正确
- 检查后端Socket.IO是否启用（应在main.ts中配置）
- 检查防火墙或代理设置

#### 问题2: 三模型调用失败
- 检查 `backend/.env.development` 中的API密钥：
  - `OPENAI_API_KEY`
  - `ANTHROPIC_API_KEY`
  - `DASHSCOPE_API_KEY`
- 检查API配额和余额
- 查看后端日志中的具体错误

#### 问题3: 数据库错误
- 运行迁移：`npm run migration:run`
- 检查 `.env.development` 中的数据库连接配置
- 确保PostgreSQL服务正在运行

#### 问题4: Redis连接失败
- 确保Redis服务正在运行
- 检查 `.env.development` 中的Redis配置
- 默认端口：6379

#### 问题5: 质量验证失败
- 三模型输出格式不一致可能导致验证失败
- 检查后端日志中的详细验证报告
- 可能需要调整prompt或validation阈值

### 性能基准
